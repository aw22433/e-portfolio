Unit 11: Pointers, References & Memory, and Design Patterns

Learning outcomes

  • Gain an appreciation of the fact that Python does not use pointers, while other programming languages, such as C, do.
  • Understand that the way in which code is design can contribute a carbon footprint and consider the design and development of code to minimise this footprint.
  • Recognise that code may be organised in a variety of manners dependent on the way in which components will interact with one another.

Artefacts

  • Summative assessment: System Implementation

  • Summative assessment: System Implementation - Feedback from Tutor

    • Reflective practice

    The primary goal for this unit was to complete the second Summative assessment: System Implementation. I found this very challenging but an excellent opportunity to bring together everything I had learned so far in this module. Unfortunately, some areas of my Design Plan were unsuitable, so I had to adjust these prior to implementing my design. As with the Project from my last module, Launching into Computer Science, I found that I learned an incredible amount by getting stuck in to programming a larger project such as this one. While building this project, I was continuously going back over previous units to bring together everything I had learned, and I was happy with the outcome. There were areas of this project which I do believe could be improved, however following the conclusion of this module I will review the feedback and the project and improve it as this will be invaluable to my ongoing progression.
