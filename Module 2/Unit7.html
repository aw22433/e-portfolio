Unit 7: Debugging/Error Handling, Data Structure and Data search

Learning outcomes

  • Gain an understanding of the need to debug code and the typical types of errors which can be found in a program.
  • Examine the ability of linters to support the development of high quality code.
  • Become familiar with a range of data structures which may be used to store data.
  • Explore the effectiveness of different search techniques to examine the stored data.

  • Feedback from Summative assessment: System Design

    • Reflective practice

    During this unit we had to submit our summative assessment on system design. I was very confident with several of the UML model types, for example the Use case diagram and the state diagram. Unfortunately, the feedback regarding the sequence diagram and class diagram was not positive. I therefore had to review both diagram types. For the class diagram, I missed some key information into the relationship between each of the object types. Furthermore, my sequence diagram was not correct. This is clearly an area which I will need to review to increase my skills in.
